Nmap Scans and TCP Port Finding with Nmap

Nmap Scans

Certainly! Here are examples of more specific Nmap scans, including TCP Connect() scan, SYN stealth scan, UDP scan, Ping scan, and Idle scan. These scans serve different purposes and are used under various circumstances based on the type of information you want to gather and the network environment.

  1. TCP Connect() Scan

    This scan is the most basic form of TCP network scanning. Instead of using raw packets, this scan uses the standard TCP/IP protocol to establish a connection.

    nmap -sT [target_ip]

    Example:

    nmap -sT 192.168.1.100

  2. SYN Stealth Scan

    SYN scan is less intrusive and is often used for stealth scanning, as it does not complete the TCP handshake.

    nmap -sS [target_ip]

    Example:

    nmap -sS 192.168.1.100

  3. UDP Scan

    This scan is used for scanning UDP ports. UDP scans are typically slower and may require more time to complete.

    nmap -sU [target_ip]

    Example:

    nmap -sU 192.168.1.100

  4. Ping Scan

    Ping scans are used to determine if the target is online without scanning the ports. This scan simply pings the target.

    nmap -sn [target_ip]

    Example:

    nmap -sn 192.168.1.100

  5. Idle Scan

    An idle scan is an advanced scan technique that uses a "zombie" host to scan a target without revealing your IP. You need to find a suitable zombie host with a predictable IP ID sequence to use this scan.

    nmap -sI [zombie_ip] [target_ip]

    Example (assuming `192.168.1.50` is the zombie host):

    nmap -sI 192.168.1.50 192.168.1.100

Additional Notes

  • Legality: Remember, unauthorized scanning is illegal. Always have permission before scanning a network.
  • Firewalls and IDS: Some of these scans can be detected by firewalls or intrusion detection systems.
  • UDP Scanning: UDP scanning is generally slower and less reliable than TCP scanning.

Each scan type has its specific use case. For instance, a TCP Connect() scan is straightforward but easily detectable, while a SYN stealth scan is more discreet. UDP scans are essential for complete network mapping but are slower. Ping scans quickly tell you about the live hosts in a network, and idle scans are used for stealthy, indirect scans.

Finding Open TCP Ports with Nmap

Certainly! Below are the step-by-step instructions on how to find open TCP ports using Nmap, along with examples of different types of scans:

  1. Step 1: Install Nmap

    Ensure that Nmap is installed on your Linux system. If not, you can install it using your package manager. For example, on Ubuntu, you can use:

    sudo apt-get install nmap

  2. Step 2: Perform Basic TCP Scan

    Run a basic TCP scan on the target IP address. This scan will scan the 1000 most common TCP ports.

    nmap [target_ip]

    Example:

    nmap 192.168.1.100

  3. Step 3: Perform SYN Scan (Stealth Scan)

    A SYN scan is stealthier than the default TCP scan and can be used to evade intrusion detection systems.

    nmap -sS [target_ip]

    Example:

    nmap -sS 192.168.1.100

  4. Step 4: Perform Service Version Detection

    This scan detects service versions running on open ports.

    nmap -sV [target_ip]

    Example:

    nmap -sV 192.168.1.100

  5. Step 5: Scan Specific Ports

    You can scan specific TCP ports or a range of ports.

    nmap -p [port(s)] [target_ip]

    Example (scan ports 80, 443, and 22):

    nmap -p 80,443,22 192.168.1.100

    Example (scan ports 1 to 1000):

    nmap -p 1-1000 192.168.1.100

  6. Step 6: Perform Aggressive Scan

    An aggressive scan includes OS detection, version detection, script scanning, and traceroute.

    nmap -A [target_ip]

    Example:

    nmap -A 192.168.1.100

  7. Step 7: Save Output to a File

    You can save the scan results to a file for future reference.

    nmap -oN [output_file] [target_ip]

    Example:

    nmap -oN scan_results.txt 192.168.1.100

These steps cover the basic usage of Nmap for finding open TCP ports on a Linux system. Experiment with these commands and explore Nmap's extensive capabilities for network scanning and security auditing.
